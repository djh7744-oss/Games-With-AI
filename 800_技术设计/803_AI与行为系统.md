# AI与行为系统架构

## 一、四层AI架构

```
战略层(国家AI) —— Utility AI + GOAP
  频率: 每月/每季决策
  决策: 宣战/结盟/变法/科技方向/资源分配
  实现: C++ 效用评估系统

战役层(军团AI) —— Behavior Tree + EQS
  频率: 每天/每次战斗决策
  决策: 军团移动路线/战术选择/进攻撤退时机
  实现: Behavior Tree + EQS查询

战术层(战场AI) —— State Tree + Mass AI
  频率: 每1-5秒
  决策: 士兵个体行为/编队协同/阵型保持
  实现: State Tree + Mass Entity(远景)

反应层(个体AI) —— Behavior Tree
  频率: 实时
  决策: NPC反应/对话/情绪
  实现: Behavior Tree + 脚本
```

## 二、势力AI实现

```cpp
// 效用评估系统
UCLASS()
class UStrategicAIComponent : public UActorComponent
{
    // 每月评估所有可能行动
    void EvaluateMonthly();
    
    // 计算行动的效用分数
    float CalculateUtility(FStrategicAction Action);
    
    // 选择最优行动组合
    TArray<FStrategicAction> SelectBestActions();
    
    // 执行选中的行动
    void ExecuteActions(TArray<FStrategicAction> Actions);
};

// 示例: 秦国评估"进攻韩国"
// Utility = 军事实力比(0.3) + 领土接壤(0.2) + 资源需求(0.15) 
//         + 外交孤立度(0.2) + 君主性格(0.15)
// = 0.85(高) → 决定进攻
```

## 三、战场士兵AI

```
士兵状态机 (State Tree):

[Idle] ──敌人接近──→ [Combat]
  ↑                      ↓
  └──敌人消失──── [Flee] ←──士气崩溃
                      ↓
                  [Rally] ──恢复──→ [Idle]

Combat子状态:
  [Approach] → 向敌人移动(保持阵型约束)
  [Attack] → 进入范围→攻击动画→伤害判定
  [Block] → 检测来袭攻击→格挡判定
  [Retreat] → 体力低/被包围→撤向友军
  [Regroup] → 溃逃后在远处重整

性能优化:
  LOD0(近景<50m): 完整State Tree, 每1-2秒决策
  LOD1(中景50-150m): 简化State Tree, 每5秒决策
  LOD2(远景150m+): Mass Entity, 编队级AI(无个体决策)
```

## 四、EQS(环境查询系统)

```
EQS用于AI评估环境:

示例: 弓兵寻找最佳射击位置
  EQS Query: "找到高地+视野好+前方有友军保护的位置"
  
  评分项:
    - 高度: 越高越好(权重0.4)
    - 到敌人距离: 在射程内(权重0.3)
    - 前方有友军: 安全(权重0.2)
    - 到最近敌人距离: 不要太近(权重0.1)
  
  → 返回最高分位置 → AI移动过去

示例: 骑兵寻找冲锋目标
  EQS Query: "找到脆弱的、侧翼暴露的敌人编队"
  
  评分项:
    - 敌人类型: 弓兵/轻步兵优先(权重0.4)
    - 侧翼暴露度: 越高越好(权重0.3)
    - 距离: 适合冲锋的50-100m(权重0.2)
    - 敌人士气: 越低越好(权重0.1)
```

## 五、Mass Entity(千人战场)

```
UE5 Mass Entity用于远景士兵:

优势:
  - CPU开销极低(无Actor开销)
  - 批量处理(ECS架构)
  - 支持千人级渲染

使用方式:
  - 近景士兵(可见细节): 标准Actor+完整AI
  - 远景士兵(远处的人海): Mass Entity
  - 过渡: Actor→Mass Entity 在50-80m平滑切换

Mass Entity数据:
  Fragment: 位置/朝向/动画帧/队伍ID
  Processor: 移动/动画更新/渲染LOD
```

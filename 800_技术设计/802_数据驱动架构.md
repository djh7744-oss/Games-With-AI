# 数据驱动架构

## 一、数据管线

```
策划(Excel/CSV) → UE5 DataTable → C++/Blueprint → 运行时
                     ↑
                     数据验证脚本(自动检查)
                     ↑
                     Git版本控制(CSV可Diff)

管线步骤:
  1. 策划在Excel中编辑数值
  2. 导出为CSV(UTF-8, 逗号分隔)
  3. CSV放入 Content/DataTables/
  4. UE5 Editor自动导入→DataTable Asset
  5. 数据验证脚本运行(检查合法性)
  6. 运行时: DataTable加载→C++结构体→游戏逻辑使用
```

## 二、核心数据结构

```cpp
// 兵种数据

    GENERATED_BODY()

    UPROPERTY(EditAnywhere) int32 Attack;

    UPROPERTY(EditAnywhere) int32 Health;

    UPROPERTY(EditAnywhere) int32 Cost_Gold;
    UPROPERTY(EditAnywhere) int32 Cost_Population;
    UPROPERTY(EditAnywhere) int32 TrainDays;

// 物品数据
USTRUCT(BlueprintType)
```

## 三、DataAsset使用

```
适合DataAsset的场景(而非DataTable):

1. 国家配置(NationConfig_DA):
   - 七国各一个DataAsset
   - 存储: 国家颜色/初始领土/初始关系/AI权重
   - 不放在DataTable → 因为结构复杂/各有嵌套

2. 玩家出身配置(OriginConfig_DA):
   - 六出身各一个DataAsset
   - 存储: 属性修正/初始装备/初始关系/专属任务链

3. 技能树配置(SkillTreeConfig_DA):
   - 每条技能树一个DataAsset
   - 存储: 树结构(节点+连接)/节点解锁条件/节点效果

4. Biome配置(BiomeConfig_DA):
   - 九大Biome各一个DataAsset
   - 存储: 地形材质/植被类型/天气概率/环境音效
```

## 四、数据验证

```
CSV导入前自动验证:

1. 引用完整性:
   □ 所有FName引用的ID在对应表中存在
   例: UnitData.RequiredTech → TechTree表中必须存在

2. 数值范围:
   □ 攻击力: 10-200
   □ 防御力: 5-150
   □ 价格 >0
   □ 训练天数 >0

3. 逻辑一致性:
   □ T5兵种的属性必须>T4>T3>...
   □ 名将的统率>普通将领
   □ 品质越高的物品价值越高

4. 命名规范:
   □ ID格式: [前缀]_[国家]_[名称]_[变体]
   □ 无中文字符(仅FText字段可用中文)

验证失败 → 阻止导入 → 导出错误报告
```

## 五、热更新支持

```
数据热更新(无需重新打包):

DataTable热重载:
  开发期: 修改CSV → UE5自动检测 → 重新导入
  运行期: 修改CSV → 游戏内控制台命令 → 重载DataTable

运行时数据修改(仅开发/测试):
  console: DataTable.Reload WarringStates.UnitData
  console: DataTable.Dump UnitData → 输出当前数据到日志

正式发布:
  热更新通过Steam/Epic更新机制 → 补丁包
  不开放运行期数据修改(防作弊)
```

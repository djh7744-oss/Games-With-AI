# 工具链与自动化

## 一、开发环境

```
引擎: UE 5.5+ (Epic Games Launcher 或 源码编译)
IDE: Visual Studio 2022 / Rider
建模: Blender 4.x LTS
贴图: Substance Painter / Quixel Mixer
音频: Wwise / Reaper(DAW)
版本控制: Git + Git LFS + Perforce(大资产可选)

推荐硬件:
  CPU: AMD 7950X or Intel i9-14900K
  RAM: 64GB
  GPU: RTX 4080+
  SSD: 2TB NVMe
```

## 二、数据导入管线

```
CSV → DataTable自动化:

Editor Utility Widget / Python脚本:
  1. 监控 Content/DataTables/Source/ 目录
  2. 检测到新CSV → 自动导入 → 生成DataTable Asset
  3. 运行验证脚本 → 检查数据合法性
  4. 验证通过 → 提交到版本控制
  5. 验证失败 → 发送报告给提交者

Blender → UE5自动化:
  导出脚本(Blender Python):
    1. 检查模型面数/命名/材质是否符合规范
    2. 导出FBX到Content对应目录
    3. UE5自动导入(Editor Utility)
    4. 自动设置碰撞/LOD/材质

批量重导入:
  美术更新贴图/模型 → 
  UE5 Editor检测到文件修改 → 
  自动重新导入所有受影响的Asset
```

## 三、关卡生成工具

```
程序化战略地图生成:

输入:
  - 高度图(Heightmap) — 基于真实SRTM数据
  - Biome遮罩图 — 手工绘制九大Biome范围
  - 城池位置表(CSV) — 策划配置

生成步骤(Python/Houdini → UE5):
  1. 导入Heightmap → 生成Landscape
  2. 根据Biome遮罩 → 分配材质/植被类型
  3. 根据城池位置表 → 放置城池Actor
  4. 生成道路网络(城池间连线)
  5. 放置资源点/关隘/渡口
  6. Procedural Foliage → 按Biome生成植被
  7. 最终 → 导出为World Partition关卡

战场关卡生成:
  输入: 战略地图坐标
  1. 查询坐标处的Biome+高度+附近特征
  2. 生成小尺寸Landscape(1-4km²)
  3. 依据Biome配置放置植被/岩石
  4. 依据"附近特征"放置建筑(村庄)/河流
  5. 标定部署区(双方)
  6. 导出战场关卡实例
```

## 四、CI/CD管线

```
持续集成(Jenkins / GitHub Actions):

每次提交触发:
  1. 编译检查 (Windows)
     - Build Development Editor
     - 如有编译错误 → 通知提交者
  2. 数据验证
     - 运行CSV验证脚本
     - 检查Asset引用完整性
  3. 自动化测试
     - 运行单元测试(Unit Tests)
     - 运行功能测试(Functional Tests)
  4. 性能基准测试
     - 在测试机器上运行基准关卡
     - 记录FPS/内存/加载时间
     - 与上次提交比较 → 性能退化>5% → 警告

每夜构建(Nightly Build):
  - 完整编译+打包(Shipping)
  - 冒烟测试(Smoke Test) → 自动游玩30分钟
  - 生成测试报告

里程碑构建(Milestone Build):
  - 打包为可分发版本
  - 上传到内部测试平台
  - 通知QA团队测试
```

## 五、调试工具

```
游戏内调试:

控制台命令(Console Commands):
  stat fps          → 显示帧率
  stat gpu          → GPU性能分析
  stat unit         → 各线程耗时
  stat scenerendering → 渲染统计
  DataTable.Reload  → 热重载数据表
  AI.Debug          → 显示AI决策可视化

UE5内置工具:
  Unreal Insights   → 详细性能跟踪
  GPU Visualizer    → GPU瓶颈定位
  RenderDoc         → 帧分析(第三方)
  Network Profiler  → 网络(预留)
  Wwise Profiler    → 音频调试

作弊码(开发期):
  God               → 无敌
  Fly               → 飞行模式
  GiveGold [amount] → 加钱
  SkipTime [days]   → 快进时间
  CompleteQuest [ID]→ 完成任务
  UnlockAllTech     → 解锁全部科技
```

## 六、版本发布流程

```
版本号规则: v[主版本].[次版本].[修订号]

主版本(Major): 重大变更(如: EA→正式版)
次版本(Minor): 新功能/内容更新
修订号(Patch): Bug修复/小调整

发布检查清单:
  □ 全部编译通过(Development + Shipping)
  □ 自动化测试全部通过
  □ 性能不退化(对比上个版本)
  □ 无崩溃Bug(P0)
  □ 数据验证通过
  □ 本地化文本已更新
  □ 平台认证(Steam/Epic)已完成
  □ 发布说明(Changelog)已撰写
  □ 已知问题列表(Known Issues)已更新
```

# 核心系统架构

## 一、UE5框架层定制

```
WarringStatesGameInstance (全局单例)
  └── WarringStatesGameMode (游戏模式)
        ├── WarringStatesGameState (全局状态: 时间/天气/世界事件)
        ├── WarringStatesPlayerController (玩家控制器)
        │     ├── InputComponent (输入绑定)
        │     ├── CameraManager (摄像机管理)
        │     └── HUD (UI管理)
        ├── WarringStatesPawn (玩家Pawn-战略地图)
        │     └── WarringStatesCharacter (玩家Character-战斗/城池)
        └── WarringStatesPlayerState (玩家状态: ID/国家/属性/爵位)

核心生命周期:
  GameInstance::Init() → 加载全局配置/语言/音频
  GameMode::InitGame() → 初始化世界状态
  GameMode::StartPlay() → 游戏开始
  GameState → 时间流逝/世界事件/国家AI逻辑
  存档 → GameInstance::SaveGame() → SaveGame Object → 磁盘
```

## 二、事件总线系统

```
全局事件总线 (Event Bus):

使用UE5 GameInstanceSubsystem:

class UEventBusSubsystem : public UGameInstanceSubsystem
{
    // 注册/注销/广播事件
    void BroadcastEvent(FGameplayTag EventTag, FEventData Data);
    
    // 常用事件:
    // "War.OnDeclared"          → 两国宣战
    // "Battle.OnStart"          → 战斗开始
    // "NPC.OnDeath"             → NPC死亡
    // "Economy.OnPriceChange"   → 物价波动
    // "Quest.OnCompleted"       → 任务完成
    // "Player.OnLevelUp"        → 玩家升级
    // "World.OnDisaster"        → 自然灾害
    // "Diplomacy.OnTreatySigned"→ 条约签署
};

子系统间通过事件总线解耦:
  战斗系统 → 发布 "Battle.OnVictory"
  → UI系统订阅 → 显示胜利界面
  → 声望系统订阅 → 计算声望变化
  → 任务系统订阅 → 检查任务完成条件
  → AI系统订阅 → 重新评估战略
```

## 三、存档系统

```
SaveGame架构:

USaveGame_WarringStates
  ├── PlayerData (玩家属性/技能/装备/背包/爵位)
  ├── WorldData (时间/天气/世界事件状态)
  ├── NationData[7] (各国国力/城池/外交/科技/军队)
  │     └── CityData[40+] (每城七维数值+NPC)
  ├── NPCRelationData (NPC好感/忠诚/关系网)
  ├── QuestData (任务状态/进度/历史)
  ├── EventHistory (已触发的事件/选择/蝴蝶效应)
  └── SettingsData (玩家设置)

存档策略:
  自动存档: 每30分钟 + 重要事件前(大战/关键选择)
  手动存档: 最多10个槽位 + 1快速存档
  存档压缩: 二进制序列化+Zlib压缩
  存档大小: 预估 ~5-15MB (取决于世界复杂度)

兼容性:
  版本号标记 → 旧存档可迁移(DataMigration)
  重大版本变更 → 旧存档标记为[不兼容]
```

## 四、时间系统

```
世界时间管理:

class UTimeManager : public UGameInstanceSubsystem
{
    // 时间流速
    float TimeScale = 1.0f;  // 1x(标准)/2x/0.5x/暂停(0)
    
    // 时间表示
    int32 Year;    // 秦王政二十年
    ESeason Season; // 春/夏/秋/冬
    int32 Month;   // 1-12
    int32 Day;     // 1-30
    int32 Hour;    // 0-23
    
    // 时间流逝比: 1现实秒 = 游戏中X秒
    float RealToGameTimeRatio = 60.0f; // 1分钟 = 1小时
    
    // 每日事件触发
    void OnNewDay();  // 物价更新/AI决策/存档
    void OnNewMonth(); // 税收/朝贡/科技进度
    void OnNewSeason(); // 季节效果/产出变化
    void OnNewYear();  // 年度结算/考核/节日
};
```

## 五、模块依赖规则

```
模块依赖图(从上到下):

UI层: WarringStatesUI (依赖所有下层)
  ↓
游戏逻辑层: Combat / Economy / Diplomacy / Strategy / Narrative
  ↓ (游戏逻辑层互相可引用)
核心层: WarringStatesCore (GameMode/GameState/存档/事件总线)
  ↓
框架层: UE5 Engine (不修改引擎代码)

规则:
  ✅ 上层可依赖下层
  ✅ 同层可互相依赖(游戏逻辑层)
  ❌ 下层不可依赖上层
  ❌ 核心层不可依赖游戏逻辑层
  ❌ 不可修改引擎代码(仅插件形式扩展)
```

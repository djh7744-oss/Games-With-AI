# 性能优化策略

## 一、帧预算分配

```
目标: 60FPS = 16.6ms/帧

CPU线程分配:
  Game Thread: 5ms (游戏逻辑/AI/物理)
  Render Thread: 4ms (DrawCall/遮挡剔除)
  Audio Thread: 2ms (Wwise)
  Other: 1.6ms (空闲)

GPU预算(6ms):
  几何(Geometry): 1.5ms (Nanite)
  光照(Lighting): 1.5ms (Lumen)
  后处理: 1.0ms
  阴影: 0.5ms (Virtual Shadow Maps)
  特效: 1.0ms
  UI: 0.5ms

30FPS降级方案(最低配置):
  Nanite: 关闭 → 传统LOD
  Lumen: 关闭 → 烘焙光照
  阴影: Virtual Shadow Maps → Cascaded Shadow Maps
  分辨率: 1080p → 900p动态分辨率
```

## 二、LOD策略

```
Static Mesh LOD:
  LOD0: 100% △ (0-20m)  — 完整模型
  LOD1: 50% △ (20-80m)  — 简化几何
  LOD2: 20% △ (80-200m) — 极简几何
  LOD3: 5% △ (200m+)   — Imposter/Billboard

Skeletal Mesh LOD (角色):
  LOD0: 100%骨骼+完整面数 (0-15m)
  LOD1: 60%面数+简化骨骼 (15-50m)
  LOD2: 30%面数+移除面部/布料 (50-100m)
  LOD3: VAT顶点动画 (100m+)

Nanite (UE5):
  用于: 建筑/地形/雕像/大型物件
  不用: 角色/Skeletal Mesh/植被(已有Procedural Foliage优化)
  效果: 自动LOD，几乎无DrawCall增加

植被LOD:
  LOD0: 完整3D树(2000-5000△) — <30m
  LOD1: 简化树(500-1000△) — 30-80m
  LOD2: Billboard(2面片) — 80-200m
  LOD3: 单Billboard — 200m+
```

## 三、内存预算

```
目标: 总内存<12GB (16GB系统) / <6GB (8GB系统)

GPU VRAM (8GB卡):
  Textures: 3GB (Texture Streaming管理)
  Geometry: 2GB (含Nanite数据)
  Render Targets: 1.5GB
  Shadow Maps: 0.5GB
  Other: 1GB

CPU RAM:
  Game Logic: 2GB (世界状态/DataTable)
  Asset Cache: 1GB
  Audio: 250MB
  UI: 100MB

Texture Streaming:
  Pool Size: 3000MB(8GB VRAM) / 2000MB(6GB VRAM)
  根据距离动态调整分辨率
  近景: 2K贴图
  中景: 1K
  远景: 512
  极远: 128
```

## 四、战场千人优化

```
1000士兵战场 → 60FPS策略:

分层渲染:
  近景士兵(0-50m, ~30人): 完整3D+完整AI+完整动画
  中景士兵(50-150m, ~200人): 简化模型+简化AI+简化动画
  远景士兵(150m+, ~770人): Mass Entity + VAT动画

Mass Entity (UE5):
  Fragment: 位置/朝向/动画帧索引/编队ID
  Processor: 批量更新位置+播放VAT动画
  Mesh: 共享的低面数Mesh (1个Mesh渲染所有远景士兵)
  CPU: 所有770人开销 < 0.5ms

动画优化:
  Animation Sharing: 同编队士兵共享动画蓝图实例
  LOD动画更新: LOD2仅每5帧更新一次
  VAT(顶点动画纹理): 远景士兵无骨骼开销

物理优化:
  近景: 完整Chaos Physics碰撞
  中景: 简化碰撞(胶囊体)
  远景: 无碰撞(纯视觉)
```

## 五、World Partition(战略大地图)

```
16km×14km战略地图:

World Partition Grid:
  Cell Size: 128m×128m
  总Cells: ~14000个
  加载范围: 玩家周围 3×3 = 9 Cells (~384m)
  
流式加载:
  玩家移动 → 动态加载前方Cells → 卸载后方Cells
  城池: 独立DataLayer(进入城池时加载)
  战场: 独立Level Instance(战斗时生成)

DataLayer(用于逻辑分组):
  DL_AlwaysLoaded: 核心地形/全局Actor
  DL_Cities: 城池建筑(按需加载)
  DL_Foliage: 植被(随Cell加载)
  DL_NPCs: NPC(仅在城池加载时)
```

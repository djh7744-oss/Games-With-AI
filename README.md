# 战国 · 乱世风云 — 游戏策划工程

> 以中国战国时期（公元前475年-前221年）为背景的开放世界沙盒游戏。

## 版本控制规范

### Git 仓库结构

```
main              ← 正式发布版本（与游戏版本同步）
  └── develop     ← 开发主线
       ├── feature/combat-v2     ← 战斗系统V2开发
       ├── feature/economy       ← 经济系统开发
       └── feature/dlc-changping ← 长平之战DLC
```

### 提交信息规范

```
类型: [GDD]/[DATA]/[NARR]/[MAP]/[ART]/[AUDIO]/[UI]/[TECH]/[REF]/[TEMPLATE]

示例:
  [GDD] 新增军团指挥系统设计文档
  [DATA] 调整魏武卒招募成本：800金→650金
  [NARR] 完成白起个人列传初稿
  [MAP] 扩展400_地图关卡，新增6个子文档
  [TEMPLATE] 新增叙事设计/关卡设计/美术规范/传记模板
```

### 文档命名规范

```
格式: [编号]_[中文名称].md / .csv
示例:
  110_战斗系统概述.md
  210_七国兵种数据总表.csv
  321_秦国人物传记.md
  401_战略大地图设计.md

编号规则: 目录编号_子项编号 (如 110 = 100(系统设计) + 10(战斗系统))
数据表编号: 210-280 (与100系统设计编号错开)
```

### 版本号规则

```
v0.x  — 草稿阶段，内容不完整
v1.0  — 初版完成，待评审
v1.x  — 评审后修改
v2.0  — 重大修订（需在文档内记录变更日志）
vFINAL — 已锁定，对应游戏内实装版本
```

## 工作流程

### 1. 新建文档

```bash
# 复制模板
cp Templates/设计文档/系统设计模板.md 100_系统设计/xxx_新系统.md

# 编辑文档（Markdown 编辑器：VS Code / Obsidian / Typora）
# 完成初稿后提交
git add .
git commit -m "[GDD] 新增XXX系统设计文档 v0.1"
```

### 2. 评审流程

```
策划写初稿 (v0.x)
    ↓
程序评审 "技术上可实现吗？"
美术评审 "资产工作量合理吗？"
    ↓
策划根据反馈修改 (v1.x)
    ↓
制作人/主策最终审批
    ↓
状态改为 approved → 进入开发排期
```

### 3. 与代码/资产的关联

```
策划文档中的每个功能点，对应:
  ├── JIRA/TAPD 任务编号
  ├── UE5 Blueprint/C++ 类名
  ├── UE5 DataTable 行名
  └── 美术资产文件路径

示例:
  魏武卒兵种 → BP_WeiEliteInfantry.uasset
              → DT_SoldierStats.csv (Row: Wei_Elite)
              → /Content/Characters/Soldiers/Wei/SK_Wei_Heavy.fbx
              → JIRA: WG-0147
```

## 关键原则

1. **文档即真相** — 游戏怎么做，文档说了算。口头约定不算数
2. **先文档后代码** — 没有审批通过的文档，不开始写代码
3. **数据驱动** — 能用 CSV/DataTable 的数值，不硬编码在文档里
4. **图文并茂** — 每份系统文档至少有一张流程图（Mermaid/ASCII）
5. **AI 辅助，人审定** — AI 生成初稿可以，最终版本必须人审过
6. **保持同步** — 文档版本和游戏版本对应，改了游戏要更新文档
7. **编号唯一** — 每个目录/文件编号全局唯一，不重复
8. **交叉引用** — 文档间通过 `related` 字段和 `[链接](路径)` 建立关联网

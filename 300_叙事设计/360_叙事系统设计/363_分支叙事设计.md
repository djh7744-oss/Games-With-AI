# 分支叙事设计

## 一、分支类型

```
线性分支: A→B→C→(选择)→D1 or D2→E(汇合)
  例: 接受任务→执行→最后选择"杀/放"→不同结局→汇合到后续

树状分支: A→(选择)→B1 or B2→C1 or C2→(永不汇合)
  例: 霸道路线/王道路线——选择后进入完全不同的剧情线

网状分支: A→B→(选择)→C1→(选择)→可能回到B→...
  例: 外交关系网——与A国结盟→与B国交恶→B国拉拢C国→你再去拉拢C国...

隐藏分支: 条件隐藏——只有满足特定条件才会出现
  例: 如果玩家智略≥85→对话中多出一个选项→触发隐藏剧情
```

## 二、分支复杂度管理

### 分支预算

```
每个任务/场景的"分支预算":

Lv2任务(小任务): 1个选择点，2个分支，分支间差异=奖励±20%
  设计工时: 0.5天

Lv3任务(中任务): 2个选择点，3-4种结果，分支间差异=后续任务不同
  设计工时: 1-2天

Lv4任务(路线里程碑): 3个选择点，4-6种结果，分支间差异=人物命运/关系
  设计工时: 3-5天

Lv5场景(史诗时刻): 4+选择点，8+种结果，分支间差异=结局不同
  设计工时: 5-10天(仅关键场景)

总分支预算(全游戏):
  Lv2: 200+ → 100天
  Lv3: 48(6路线×8里程碑) → 约80天
  Lv4: 12(关键选择) → 约50天
  Lv5: 6(路线结局) + 4(历史分支) → 约80天
  总计: ~310天叙事设计工作量
```

### 组合爆炸管理

```
问题: 6条路线×10个关键选择×4个选项×...=天文数字的组合
      不可能手工为每种组合写独特内容

解决方案——"关键差异+填充共通":

1. 识别"真正产生差异"的节点:
   哪些选择会导致后续剧情"质变"？
   例: 荆轲是否去刺秦 → 质变(角色可能死亡)
       你选择帮助哪个国家 → 质变(后续剧情不同)
       你在某个任务中多赚了100金 → 不是质变

2. 对于"非质变节点": 使用模板化文本+参数差异
   "你帮助了[NPC名]，获得了[金额]金和[声望类型]+[数值]"
   文本框架相同，参数不同

3. 对于"质变节点": 手工写作独特内容
   每个质变节点 = 1个"叙事章节" = 专属文本/场景/后果

质变节点统计(全游戏): 约40-50个
  = 40-50个"叙事章节"需要手工打磨
  = 每个约1-3天工作量
  = 可控的设计范围
```

## 三、分支的可追溯性

```
玩家应该能"看到"自己选择带来的后果:

短期反馈(立即):
  做出选择 → NPC立即做出反应 → 世界状态立即微调
  "你说服了赵王。赵王决定不换廉颇。"
  → 廉颇好感+15，赵括不会出现在前线

中期反馈(数小时/数天后):
  之前的某个选择 → 现在出现了它的后果
  "因为你在长平之战中释放了降卒 → 
   现在一个赵国老兵找到你 → 他的儿子在你的军队中——自愿入伍"

长期反馈(数年后/结局时):
  结局CG中展示"蝴蝶效应链"
  "因为你选择了XXXX → 导致了YYYY → 最终ZZZZ"
  
  例: "公元前260年，你在长平释放了赵卒。
       公元前228年，秦灭赵时，赵国抵抗微弱——因为赵人还记得。
       你的仁义，让统一少流了十万人的血。"
```

## 四、假选择 vs 真选择

```
假选择(False Choice):
  选项不同但结果相同
  "你选择用剑还是用刀?"→无论选什么都斩杀敌人
  ❌ 坏设计: 让玩家以为有选择，其实没有
  
真选择(True Choice):
  选项不同→结果不同→后续不同
  "你选择杀了他/放了他/招募他?"
  → 杀: 得到装备，与该国关系恶化
  → 放: 仁义+，他日后可能报恩
  → 招募: 获得一个强大的部下，但原雇主仇恨你

所有的Lv4/Lv5选择必须是真选择。
Lv2/Lv3可以有假选择(不影响实质，但增加沉浸感)。

"假选择"的正确用法:
  不是"骗玩家"，而是"让玩家表达态度"
  例: "你觉得这场战争值得吗?"
      选项A: "值得。为了统一。"
      选项B: "不值得。代价太大了。"
      选项C: "我不知道。"
      
      三个选项不会改变战争结果(战争已经发生了)。
      但会改变NPC对你的看法——你是一个"理想主义者"还是"现实主义者"？
      这会微妙地影响后续NPC与你的互动。
```

## 五、分支文档标准

```
每个Lv4+分支需要以下文档:

分支节点文档模板:
  ## 节点: [节点名称]
  ### 位置: [在哪条路线/哪个任务/哪个章节]
  ### 前置条件: [触发此节点的条件]
  ### 选项:
    A. [选项文本] 
       → 后果: [详细描述]
       → NPC反应: [关键NPC的反应]
       → 世界影响: [数值变化/关系变化/事件变化]
       → 后续解锁: [解锁什么]
       → 后续锁定: [锁死什么]
    B. [选项文本] ...
  
  ### 无选项(沉默)的后果:
      [如果玩家不选，默认行为]
  
  ### 设计意图: [这个选择要测试玩家的什么价值观/能力?]
  
  ### 蝴蝶效应: [这个选择可能引发的长期连锁反应]
```

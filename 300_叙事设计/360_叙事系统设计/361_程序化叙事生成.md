# 程序化叙事生成

## 设计目标

> 79个历史将领、数百个NPC、无数随机事件——不可能手工为每一个写故事。程序化叙事系统利用模板+参数填充+世界状态检测，让游戏自己"写"出合理、连贯、有情感的小故事。这些故事也许不如手工写作的精致，但它们"真实"——因为它们是世界状态的真实反映。

---

## 一、叙事模板系统

### 模板结构

```
叙事模板 = 叙事类型 + 槽位(Slots) + 填充规则 + 情感基调

示例模板: NPC死亡通知
  类型: 个人悲剧
  槽位: [死者姓名/身份/死亡原因/死亡地点/与玩家的关系/遗言(可选)]
  填充规则:
    - 死者从玩家的"关系网络"中选取(好感≥30)
    - 死亡原因基于世界状态(战争/瘟疫/暗杀/自然死亡)
    - 遗言基于死者性格+与玩家关系生成
  情感基调: 悲伤/怀念/遗憾/愤恨(基于死亡原因)

模板实例(填充后):
  死者: 郑安平(赵括的副将，玩家长平之战中结识)
  身份: 赵国老兵，退伍后在邯郸开了一家小酒馆
  死亡原因: 邯郸保卫战期间，秦军攻城，流矢击中
  与玩家关系: 好感65(玩家曾在长平之战中救过他)
  遗言: "告诉[玩家名]...我欠他的那条命，还了。"
  
  生成的通知文本:
  "邯郸来了一封信。信封上沾着血迹。
   郑安平死了。邯郸保卫战时，他在城墙上送饭给守军——
   一支流矢。他最后说: '告诉[玩家]，我欠他的那条命，还了。'
   你记得他。长平之战时，你替他挡过一刀。
   现在他替你死了。"
```

## 二、世界状态到叙事的映射

### 战争状态 → 叙事

```
战争爆发:
  微叙事: 酒馆里"听说秦军打过来了"
  小叙事: 逃难者任务生成(护送难民/寻找失散的家人)
  中叙事: 征兵任务(某个NPC好友被征召→你可以帮他/替他)
  
战争持续:
  微叙事: 市集物价上涨("粮价又涨了...")/征兵告示
  小叙事: 军需物资任务(收集粮草/武器/药品)
  中叙事: 某个朋友的阵亡通知
  
战争结束(胜):
  微叙事: "我们赢了!"/庆祝活动
  小叙事: 凯旋任务(护送战利品/安置退伍士兵)
  中叙事: 论功行赏(你的部下获得封赏——或没有——引发情绪)
  
战争结束(败):
  微叙事: 沉默/白幡/哭泣
  小叙事: 重建任务/寻找失踪者的任务
  中叙事: 反思叙事——"我们为什么会输?"→触发政治/军事改革的动机
```

### 灾荒状态 → 叙事

```
灾荒初期:
  微叙事: "今年收成不好..."/"粮价涨了一倍"
  小叙事: 囤粮任务 vs 赈灾任务(道德选择)
  
灾荒持续:
  微叙事: 城门外出现饥民/"听说隔壁村饿死了人"
  小叙事: 处理饥民暴动/寻找粮食来源
  中叙事: 某个NPC因为灾荒做出了极端选择(卖儿卖女/偷盗/自杀)
          玩家可以干预——或不干预
  
灾荒结束:
  微叙事: "终于下雨了..."
  小叙事: 重建农业任务
  中叙事: 灾荒的教训——"我们要修水利"(触发都江堰灵感)
```

## 三、关系驱动的叙事

### NPC关系网络自动生成故事

```
系统持续监测玩家与NPC的互动，当满足条件时自动生成叙事:

"重逢"叙事:
  条件: 与某个NPC的好感曾经≥60 + 超过3年未见面 + 双方都还活着
  触发: 在某个城池偶然遇到
  文本: "你在临淄的市集上看到了一个熟悉的身影。
         [NPC姓名]。三年没见了。
         他老了一些。你也是。
         他笑着走过来——'好久不见。喝酒去?'"
  
"反目"叙事:
  条件: 好感从≥50降至≤-20(通常因为玩家的选择伤害了NPC)
  触发: NPC主动来找玩家——或不再来找
  文本: "[NPC]站在你的门外。他没有进来。
        '我听说你要和秦国结盟。'
        '是的。'
        '秦国杀了我的父亲。'
        沉默。
        '...我以为你是我的朋友。'
        他走了。没有回头。"
        
"托付"叙事:
  条件: NPC寿命将尽(年龄>寿命上限的90%) + 好感≥80
  触发: NPC主动找玩家做"最后的请求"
  文本: "[NPC]知道自己时日无多了。
        '我没什么可留给你的。只有这个——'
        他递给你[一件遗物]。
        '替我保管。或者...替我交给[某某人]。'
        他的手在颤抖。不是因为老——是因为还有很多事没做完。"
```

## 四、叙事一致性管理

### 避免"叙事矛盾"

```
问题: 程序化生成的叙事可能与手工叙事冲突
      例: 程序化生成了"你的好友张三在战场上英勇牺牲"
           但手工叙事中张三必须在三年后作为关键NPC出场

解决方案——叙事标记系统:
  每个NPC有"叙事保护标记":
    - 🔒 锁定: 核心叙事NPC = 手工叙事专用，不参与程序化生成
    - ⚠️ 受限: 重要NPC = 可参与程序化叙事，但有"幸存保证"（不会死亡/背叛）
    - ✅ 自由: 普通NPC = 完全参与程序化叙事，可以被随机事件影响
    - 💀 可牺牲: 背景NPC = 可以被程序化叙事"杀死"以增加世界真实感

  标记可随游戏进程改变:
    某个NPC完成其手工叙事后 → 保护标记从🔒降为⚠️或✅
    某个NPC成为玩家的重要伙伴 → 保护标记从✅升为⚠️
```

### 叙事历史记录

```
系统维护每个玩家角色的"叙事历史记录":

记录内容:
  - 所有程序化生成的叙事(文本+触发条件+时间)
  - 所有手工叙事的选择与结果
  - NPC命运变化(结识/绝交/死亡/背叛/和解)
  - 玩家的重大成就和重大失败

用途:
  1. 避免重复生成相似叙事(同一个"朋友牺牲"的故事不会连续发生)
  2. 生成"回顾性叙事": 五年后系统自动生成"你回想起五年前..."
  3. 作为"叙事日志"(角色日记)的素材来源
  4. 结局时生成"你的一生"总结——基于叙事历史
```

## 五、叙事质量评估

### 自动检测

```
每次程序化生成叙事 → 系统自动评估质量:

检测项:
  ✅ 逻辑一致性: 叙事中的事实与世界状态是否一致?
     (例: 不能写"秦军在邯郸大胜"——如果邯郸保卫战赵国赢了)
  ✅ 角色一致性: NPC的行为是否符合其性格?
     (例: 忠义型NPC不应该因为"对方出价高"就背叛)
  ✅ 情感连贯性: 同一NPC的连续叙事是否有情感弧线?
     (不是"周一开心/周二悲伤/周三又开心"的随机波动)
  ✅ 避免重复: 与叙事历史比对→避免生成相似度过高的叙事

评估结果:
  ⭐⭐⭐ 优秀: 可以用于游戏
  ⭐⭐ 合格: 可以使用但标记为"不突出"
  ⭐ 不合格: 生成失败→重新生成或使用备用模板
```

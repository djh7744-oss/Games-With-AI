# 叙事系统设计

## 设计目标

> 叙事不是"在游戏做完之后加上文字"。叙事系统是游戏设计的一部分——它定义了玩家如何在这个世界中"遇到故事"、"参与故事"、"改变故事"。从程序化生成的酒馆闲谈到手工打磨的历史CG，叙事系统覆盖了从微小的世界细节到改变历史进程的史诗时刻。

---

## 一、叙事的四个层级

```
层级          范围              技术手段           设计师工作量
──────────────────────────────────────────────────────
Lv1 微叙事    个体NPC的生活片段   程序化文本生成     低(模板+变量)
Lv2 小叙事    单个任务的故事      模板+参数填充      中(模板设计)
Lv3 中叙事    人物弧线/路线剧情   手工写作+分支      高(关键节点)
Lv4 宏叙事    史诗时刻/历史转折   完全手工+CG+配音   极高(仅关键场景)
```

### Lv1: 微叙事 —— 世界会说话

```
"微叙事"让玩家感觉到"世界是活的"：

示例:
  ├── 酒馆老板: "昨天来了个客人，说秦军要打过来了...希望是谣言。"
  ├── 路边老农: "我家老三去当兵了。三年没消息。他娘天天哭。"
  ├── 学宫学生: "我不同意孟子的说法! 人性未必本善..."
  ├── 城门卫兵: (打哈欠)"这班还要值三个时辰..."
  └── 市集商贩: "临淄的盐比这里便宜三成——可惜商路最近不太平。"

设计原则:
  1. 每个NPC都有1-3句"状态相关"的微叙事台词
  2. 台词随世界状态变化: 战争期/和平期/灾荒期 → 对话内容不同
  3. 玩家行为会影响微叙事:
     你昨天帮了某个NPC → 今天酒馆里有人谈论"那个好人"
     你屠了一座城 → 附近的NPC提到你时"声音颤抖"
  4. 微叙事之间偶尔产生"联系":
     酒馆老板说的"客人" = 城门口经过的商人NPC
     老农的"老三" = 战场上某个可以使用的小兵NPC
     
实现: 模板变量 + 世界状态检测 + 程序化文本合成
```

### Lv2: 小叙事 —— 每个任务都有理由

```
"小叙事"让任务不只是"去A杀B回来领赏"：

对比:
  无叙事: "护送商队从大梁到临淄。奖励800金。"
  有叙事: "李氏商行的李老板找到你。他的独子第一次押运商队，
          走的是齐魏商路。李老板担心儿子出事——
          '这孩子太实诚了，遇到山贼不会跑。壮士，帮我看顾着他。'
          路上你发现李少爷确实很天真——
          和山贼讲道理、给乞丐分干粮、还给你讲他喜欢的姑娘...
          到了临淄，他硬要多给你200金。你收不收?"

小叙事要素:
  1. 任务发布者是有名字/性格/动机的"人"
  2. 任务过程中有"小事件"——不是连续战斗，是人物互动
  3. 完成后有"反馈"——不是"任务完成"弹窗，是NPC的道谢/感谢信/后续来访
  4. 任务中的选择影响后续:
     收了李少爷的钱 → 他以后会继续找你 → 但觉得你是"可以被钱收买的人"
     不收钱 → 他欠你一个人情 → 以后会有意想不到的回报

实现: 任务模板 + NPC池 + 叙事事件库
```

### Lv3: 中叙事 —— 人物弧线

```
"中叙事"是角色的成长故事:

六条人生路线 = 六个"人物弧线"
  霸道: 从士兵到帝王的弧线 → 核心张力: 权力 vs 人性
  王道: 从策士到天下共主的弧线 → 核心张力: 理想 vs 现实
  侠道: 从游侠到侠之大者的弧线 → 核心张力: 自由 vs 责任
  商道: 从货郎到幕后之王的弧线 → 核心张力: 财富 vs 意义
  匠道: 从学徒到改变世界的弧线 → 核心张力: 技艺 vs 影响
  隐道: 从学子到百世之师的弧线 → 核心张力: 孤独 vs 传承

每个弧线包含:
  3个阶段 → 每个阶段有情感主题
  例: 霸道
    阶段1: "野心" → 我为什么要当兵? (活下去/出人头地/家族期望?)
    阶段2: "代价" → 我失去了什么? (战友/良心/家人?)
    阶段3: "意义" → 我得到了天下，然后呢? (空虚/满足/责任?)

实现: 手工写作关键节点 + 程序化填充过渡内容
```

### Lv4: 宏叙事 —— 史诗时刻

```
"宏叙事"是玩家永远不会忘记的场景:

全游戏的宏叙事场景 (约15-20个):
  ├── 荆轲: "风萧萧兮易水寒"
  ├── 长平: 白起站在四十万降卒前
  ├── 称帝: 统一天下的那一刻
  ├── 负荆请罪: 廉颇跪在蔺相如门前
  ├── 屈原投江: 最后一个背影
  ├── 商鞅被车裂: "我制定的法律...困住了我自己"
  ├── 马陵道: 火光映着"庞涓死于此树之下"
  ├── 吕不韦: "今力田疾作，不得暖衣余食。"
  ├── 鬼谷: 找到了传说中的鬼谷
  ├── 统一: 最后一座敌国城池投降
  └── ... 

每个宏叙事场景:
  ├── 专属CG或高质量实时过场
  ├── 不能跳过(首次观看)
  ├── 玩家的选择会影响场景细节
  │   例: 统一场景 → 你的声望影响百姓的反应:
  │        仁义高→万民欢呼 / 仁义低→百姓恐惧沉默
  └── 场景后 → 游戏世界永久改变 → 很多小叙事会引用这个时刻

实现: 完全手工制作 → 电影级过场 or UE5 Sequencer实时渲染
```

## 二、叙事生成系统

### 世界状态到叙事的映射

```
游戏世界中的每一个变化 → 都应该"被叙事化":

战争爆发 → 
  微叙事: 酒馆里人在讨论/农民在担忧
  小叙事: 逃难者任务/军需商人任务生成
  中叙事: 玩家部下的亲属来信/担心/请求

灾荒 → 
  微叙事: 城门外的饥民/市集上的粮价
  小叙事: 赈灾任务/投机倒把的商人
  中叙事: 某个灾民的真实故事(程序化生成)

名人死亡 → 
  微叙事: NPC对话提及
  小叙事: 该名人的门徒/后代寻求帮助
  中叙事: 纪念活动/传记撰写

玩家重大成就 → 
  微叙事: 酒馆有人唱你的歌
  小叙事: 慕名而来者/模仿者/挑战者
  中叙事: 你的家乡为你立碑/你的名字被写入史书
```

### 叙事日志系统

```
"角色日记"不是任务清单 → 是你在战国时代的"人生记录"

日记格式:
  ┌─────────────────────────────────────────────┐
  │ 📜 秦王政二十年 秋 九月十五                   │
  │                                             │
  │ 今天在邯郸城外遇到了李氏商行的李少爷。         │
  │ 他第一次押运商队，紧张得手都在抖。             │
  │ 我护送他从大梁到临淄。                       │
  │                                             │
  │ 路上遇到了小股山贼——李少爷居然上去跟           │
  │ 山贼讲道理。山贼头子是个逃兵，说家里           │
  │ 老母病了才落草。李少爷给了他十两银子。         │
  │ 山贼头子哭了。说以后再也不劫李氏商行。         │
  │                                             │
  │ 这傻小子。但傻得让人有点羡慕。                 │
  │                                             │
  │ 到临淄后他多给了我二百金。我收下了。           │
  │ 他笑着说"下次还找你"。大概会吧。               │
  │                                             │
  │ 晚上在临淄酒馆听说——秦军攻韩了。               │
  │ 宜阳被围。如果宜阳失守...                      │
  │ 算了。明天再说。今天累了。                     │
  └─────────────────────────────────────────────┘

日记的生成:
  85% 程序化: 基于任务模板+世界事件+NPC互动
  10% 半手工: 关键事件 → 预写的"日记模板" + 玩家行为插入
  5% 纯手工: 史诗时刻 → 完全手工写作的日记篇章

日记的价值:
  1. 回顾: 玩家可以看到自己"在战国活过的每一天"
  2. 情感: 多年后翻看 → "原来那时候..."
  3. 社交: 玩家可以把日记导出/截图分享 → "看看我在战国的故事"
```

## 三、叙事与游戏性整合

```
"故事"和"玩法"不是两条平行线——它们必须"咬合"在一起:

正面案例:
  孙膑的"围魏救赵" → 这是一个战术(玩法)也是一个故事(叙事)
  玩家在使用这个战术时 → 自己体验到了孙膑当年的处境
  玩法 = 叙事 = 教学

负面案例:
  "打完怪→播过场→打完怪→播过场" 
  叙事和玩法脱节 → 玩家觉得"过场是在打断我玩游戏"

咬合原则:
  1. 重要的叙事时刻 = 玩家做出游戏性选择的结果
     不是"NPC对你说教" → 而是"你做的选择导致了这件事发生"
     
  2. 叙事提供"为什么做" → 玩法提供"怎么做"
     任务: 护送李少爷 
     叙事告诉你: 李少爷是个好人，但太天真，可能会出事
     玩法: 你护送他，用战斗/魅力/智力应对各种危机
     叙事跟进: 李少爷感谢你/学到了/成长了
     
  3. 叙事"滞后"于玩法
     玩家先做出选择 → 然后叙事系统"解释"这个选择的意义
     而不是: 叙事"提前告诉"玩家应该怎么选
```

## 四、玩家叙事主权

```
"这是你的故事，不是我的故事。"——设计师的克制

设计师的角色:
  搭建舞台 (世界背景/人物/NPC/历史事件)
  设置道具 (武器/装备/金钱/领土)
  设计冲突 (战争/外交/内政/人际关系)
  然后 → 退后一步 → 让玩家自己去"演"

玩家叙事主权体现在:
  1. 玩家决定"我是谁" → 出身选择 → 路线选择 → 关键道义选择
  2. 玩家决定"我做什么" → 接哪些任务/放弃哪些/怎么完成
  3. 玩家决定"我怎么评价" → 
     日记是第一人称 → 玩家可以选择"语气"(客观/热血/悲观/嘲讽)
  4. 玩家决定"故事的结局" → 没有"官方结局" → 
     你的结局就是你的结局

设计师不应该做的事:
  ✗ 强制玩家同情某个NPC
  ✗ 只有一条"正确"的道德选择
  ✗ "你应该感到悲伤/愤怒/快乐"
  ✗ 结局的"好坏"由设计师评定
  
设计师应该做的事:
  ✓ 给出"让人纠结"的选择
  ✓ 每个选择都有合理的理由
  ✓ 让世界对玩家的选择做出"反应"
  ✓ 结局的"好坏"由玩家自己评定
```

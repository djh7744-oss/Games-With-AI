# 人物传记系统

## 设计目标

> 79位历史将领，79段人生。每篇传记不是属性罗列，而是"这个人为什么成为了这个人"——他们的出身、他们的抉择、他们的荣光与悲剧。传记同时服务于叙事设计（为剧情提供人物动机）、角色系统（为招募和关系提供背景）和AI系统（为NPC行为提供性格依据）。

---

## 传记格式标准

```
每篇传记包含以下八个维度:

1. 基本信息
   姓名/生卒年/国籍/最高官职/历史地位(一句话)

2. 出身与早年
   家族背景/少年经历/如何进入历史舞台

3. 性格画像
   三言两语勾勒性格核心→不是标签而是矛盾体
   例: 白起——"战场上的冷血杀神，私下里的沉默寡言者"

4. 关键人生节点
   3-5个改变命运的转折时刻
   每次转折: 面临什么选择→做了什么→后果是什么

5. 人际关系网
   恩师/挚友/对手/仇敌/君主关系

6. 历史功过
   时人评价/后世评价/客观分析

7. 在游戏中的呈现
   出现场景/可触发事件/招募条件/专属剧情

8. 经典语录
   1-3句最能代表此人的话(史书记载或合理创作)
```

## 传记叙事原则

```
1. 矛盾出性格
   不写"白起很能打"
   而要写"白起在坑杀40万赵卒的那天晚上，回到了营帐，
         他的副将看到他坐在黑暗中，没有说话。
         从那以后，白起再也没有笑过。"
   好的传记展示矛盾，而非罗列事实。

2. 选择见人性
   每篇传记聚焦人物面临的关键选择——
   商鞅选择得罪所有贵族推行变法，
   蔺相如选择忍让廉颇以保赵国团结，
   荆轲选择用生命去换一个虚无缥缈的成功概率。
   这些选择比属性和战绩更能定义一个人。

3. 留白给玩家
   传记不写满。留出空间让玩家在游戏中"遇见"这个人物时
   自己形成判断。传记是引子，游戏体验是正文。

4. 历史与架空交织
   基于史实，但在游戏允许的架空空间内提供"另一种可能"——
   如果赵括赢了长平之战会怎样？
   如果荆轲成功了历史会怎样？
   传记中暗示这些可能性，但不展开。

5. 游戏性关联
   每篇传记末尾明确标注→此人在游戏中的价值
   (可学什么技能/可触发什么任务/可解锁什么结局)
```

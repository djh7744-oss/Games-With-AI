# UI/UX设计概述

## 设计愿景

> "信息在手边，世界在眼前。" UI不是挡在玩家和战国世界之间的玻璃墙——它是一个窗口。HUD最小时几乎不可见（沉浸模式），需要时信息清晰可辨。每一次点击、每一次滑动，都应该像翻开一卷竹简一样自然。

## 六大UX原则

| 原则 | 含义 | 检验标准 |
|------|------|---------|
| **信息分层** | 核心信息一眼可见，详细信息展开可得 | 5秒内能否了解当前最关键的状态? |
| **最少操作** | 常用功能不超过3次点击 | 征兵→选择城池→确定: 3步完成 |
| **反馈即时** | 每个操作<100ms给出视觉+音频反馈 | 点击按钮是否有"按下了"的感觉? |
| **一致性** | 相同功能在全部界面使用相同的交互模式 | 关闭按钮永远在右上角 |
| **可逆性** | 重要操作可撤销(短期) | 卖错装备→30秒内可回购 |
| **沉浸优先** | UI可最小化，让世界本身成为界面 | 探索模式→HUD完全隐藏 |

## 全局交互规范

### 输入

```
键鼠:
  左键: 选择/确认/拖拽
  右键: 上下文菜单/取消/返回
  滚轮: 滚动/缩放
  Esc: 关闭当前面板/打开系统菜单
  Tab: 切换面板/切换地图图层
  Space: 暂停/跳过
  1-9: 快捷栏/编队选择
  F1-F6: 军团指令(战斗)/功能面板(战略)

手柄(Xbox布局):
  A: 确认
  B: 取消/返回
  X: 上下文操作
  Y: 切换视角/面板
  LB/RB: 切换Tab
  LT/RT: 缩放
  左摇杆: 移动/导航
  右摇杆: 视角/光标
  Start: 系统菜单
  Select: 地图/日志
```

### 导航

```
面包屑导航: 
  始终显示当前位置层级
  例: 角色面板 > 装备 > 武器 > 精铁剑

快速跳转:
  Ctrl+数字键: 打开对应面板
    1=角色  2=背包  3=任务  4=地图  5=外交  6=内政

返回行为:
  Esc: 逐层返回(先关闭子面板→再关闭父面板)
  Backspace: 返回上一界面(保留数据)
```

### 响应式

```
分辨率断点:
  ≥2560×1440 (2K+): 大UI模式
  1920×1080 (1080p): 标准UI
  1600×900: 紧凑UI
  1280×720: 最小UI(图标缩小，文字简化)

UI缩放: 0.8x ~ 1.5x 用户自调
宽高比: 16:9(标准) / 16:10 / 21:9(超宽HUD居中)
```

## 信息架构

```
一级面板(全局):
  角色(C) / 背包(I) / 任务(J) / 地图(M) / 外交(D) / 内政(G) / 系统(Esc)

二级面板(上下文):
  装备详情(双击装备) / 城池详情(点击城池) / 战斗部署(战前) / 
  谈判界面(外交中) / 锻造界面(铁匠铺)

三级面板(细节):
  装备对比 / 属性历史 / 任务链详情 / NPC关系图

模态弹窗:
  确认操作("确定要XXX吗?") / 重要通知(紧急事件) / 奖励领取
```

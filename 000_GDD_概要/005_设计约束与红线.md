# 设计约束与红线

## 一、技术约束

| 约束 | 原因 | 影响 |
|------|------|------|
| 单机为主 | 首发聚焦单机体验 | 无网络同步/无防作弊需求 |
| UE 5.5+ | 使用Nanite/Lumen/Mass Entity | 最低配置GTX 1060 |
| 千人战场60FPS | 核心卖点 | 需要严格的LOD和Mass Entity |
| 16×14km大地图 | World Partition流式加载 | 不能所有城池同时加载 |
| 无程序化地形(基础) | 地形依据真实SRTM数据 | 九大Biome手工调优 |

## 二、设计红线(不可逾越)

### 自由人生红线
- ❌ 不能让某条路线成为"唯一最优解"
- ❌ 不能让游侠出身的人必须先去从军
- ❌ 不能让商人出身的人无法获得社会尊重
- ❌ 不能有"堵死"的情况——任何时候至少2条可选路径

### 沙盒世界红线
- ❌ 不能有"等待玩家触发"的世界事件
- ❌ 不能让AI国家永远保持初始边界
- ❌ 玩家不能同时是所有国家的盟友

### 历史感红线
- ❌ 不能出现三国/明清时期的元素
- ❌ 不能为了"好看"完全脱离史实(魔幻铠甲)
- ❌ 架空路线需要逻辑推导(不是魔法)

### 战斗手感红线
- ❌ 不能把战斗变成"数值对撞"
- ❌ 玩家个人操作水平必须显著影响战斗结果

## 三、设计优先级(冲突时裁决)

```
1. 自由人生 > 历史感
   如果历史事实限制玩家选择自由度 → 优先自由度
   例: 历史上女性不能从军 → 游戏里可以

2. 战斗手感 > 策略深度
   如果复杂策略牺牲了即时反馈 → 简化策略
   例: 阵型10种但常用3种

3. 沙盒世界 > 所有
   如果某个功能需要"世界为玩家暂停" → 重新设计
   例: 玩家在战役中，外部时间仍然流逝

4. 性能 > 视觉
   如果美术细节导致掉帧 → 优化美术而非降低玩法

5. 深度 > 广度
   宁可10个系统做深，不要50个系统做浅
```

## 四、不做(Out of Scope)

```
首发不做:
  ✗ 多人模式 (PvP/Co-op)
  ✗ 完整语音配音(仅关键场景配音)
  ✗ 程序化剧情生成(AI写故事)
  ✗ VR模式
  ✗ 移动端版本
  ✗ 用户自制MOD支持

DLC考虑:
  ○ 多人模式
  ○ 地图编辑器
  ○ 跨平台联机

永远不做:
  ✗ 抽卡/开箱
  ✗ Pay-to-Win
  ✗ NFT/区块链
```

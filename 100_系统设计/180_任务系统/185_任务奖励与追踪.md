# 任务奖励与追踪系统

## 设计目标

> 完成任务后的奖励不是"领了就走"的收据确认——奖励应该有仪式感、有后续影响、有叙事重量。任务追踪不是冷冰冰的任务列表——而是一本有温度的人生日记，记录你在战国乱世走过的每一步。

---

## 第一部分：任务奖励系统

### 1.1 奖励类型矩阵

| 奖励类型 | 即时/持续 | 确定性/概率 | 来源任务 | 特殊说明 |
|---------|----------|-----------|---------|---------|
| **金币** | 即时 | 确定 | 大多数任务 | 基准奖励+难度加成 |
| **声望** | 即时 | 确定 | 国家/官府/义举 | 四维声望独立变化 |
| **经验/属性** | 即时 | 概率触发 | 人生路线/战斗/学术 | 成长概率公式 |
| **装备/物品** | 即时 | 概率(稀有)/确定(普通) | 讨伐/探索/锻造 | 品质随机 |
| **称号/头衔** | 永久 | 确定 | 人生路线里程碑 | 佩戴有BUFF |
| **爵位/官职** | 长期 | 确定 | 国家任务/军功 | 升迁或降级 |
| **NPC好感/关系** | 长期 | 确定 | NPC相关任务 | 影响后续互动 |
| **技能点** | 永久 | 确定 | 人生路线/学派 | 关键时刻奖励 |
| **封地/领土** | 永久 | 确定 | 国家/军功任务 | 最大奖励 |
| **情报/知识** | 信息型 | 确定 | 侦查/学术 | 无形但重要的奖励 |
| **独特剧情** | 一次性 | 确定 | 隐藏/路线任务 | 不可替代的叙事体验 |

### 1.2 奖励呈现方式

```
奖励仪式(按任务重要性分级):

普通任务(★1-2):
  快速结算界面: 任务完成→简要结果→奖励清单→关闭
  约3秒完成

重要任务(★3-4):
  叙事结算界面: NPC对话→交付任务→NPC反应→领取奖励
  约10秒+可跳过

史诗任务(★5/人生路线里程碑):
  专属CG/插图: 任务完成→特殊画面或动画→叙述性总结
  不可跳过(仅首次)

终极任务(路线完成/统一):
  全屏影片级结算: 专属成就动画→回顾你的旅程→展示对世界的影响
  约30-60秒→可跳过
```

### 1.3 奖励的后续影响

```
奖励不是终点:

金币 → 用于投资/征兵/装备 → 推动经济发展
声望 → 影响NPC态度/外交 → 改变社会地位
属性 → 解锁新技能/任务 → 角色成长
装备 → 改变战斗风格 → 实力提升
领土 → 增加收入/人口 → 国运改变
NPC关系 → 触发后续个人故事 → 关系网扩展

每项奖励都是下一个循环的起点
```

## 第二部分：任务追踪系统

### 2.1 任务日志

```
任务日志不是"待办清单"——而是"角色日记":

记录内容:
  ├── 接受日期(游戏时间)
  ├── 任务来源(在哪里接的/谁给的)
  ├── 任务背景(为什么会接到)
  ├── 当前进度(目标完成情况)
  ├── 已做出的关键选择(分支记录)
  ├── 涉及的关键NPC(可点击跳转)
  └── 我的备注(玩家可自行添加笔记)

日志风格:
  以第一人称视角记录
  示例:
    "秦王政二十三年秋，于咸阳酒馆接到护送任务。
     商贾李氏托我护送一批丝绸至临淄，酬金800。
     我带着5名随从出发，途经函谷关时遭遇小股山贼。
     击退山贼(击杀3人, 随从轻伤1人)。
     五日后抵达临淄，李氏满意，额外赏了200金。
     在临淄听说齐国正在招募客卿..."
     
  这种叙事风格让玩家回顾时仿佛在读自己的冒险故事
```

### 2.2 任务地图标记

```
大地图任务标记系统:

标记类型:
  📍 当前追踪任务目标(金色闪烁)
  📍 任务起点(来源NPC/地点)
  📍 任务路径(虚线连接相关地点)
  📍 可接任务(白色问号→鼠标悬停显示任务类型)
  ❗ 紧急事件位置(红色感叹号)
  ✔️ 已完成在此交付(绿色勾)
  ❌ 失败/过期(灰色叉)

标记筛选:
  按任务类型: 仅显示国家任务/动态任务/隐藏任务
  按区域: 仅显示当前区域可完成的任务
  按时间: 仅显示即将到期的任务
  全部显示: 默认
```

```
成就与统计面板:

  总接受: 247

按类型分布:
  动态任务: 182完成 (最常见)

  人生路线: 6条×完成度(霸道85%/王道40%/侠道60%/商道55%/匠道30%/隐道20%)
  隐藏任务: 8/30发现

任务时间轴:
  在"人生回顾"界面按时间轴排列所有已完成任务

  这是你在这个世界活过的证据
```

### 2.4 任务推荐系统

```
基于玩家当前状态推荐合适的任务:

评估因素:
  ├── 当前位置(推荐附近的→减少赶路时间)
  ├── 当前属性(推荐能力匹配的→不会太难或太简单)
  ├── 当前人生路线(推荐能推进路线的→主线引导)
  ├── 未完成任务的时间(优先推荐快过期的)
  ├── 奖励需求(缺钱→推荐高报酬/缺声望→推荐声望任务)
  └── 玩家偏好(记录玩家常接的任务类型→优先推荐)

推荐呈现:
  任务日志侧边"推荐任务"区域
  地图上标注"推荐"(金色星标)
  酒馆老板会说"最近有个活儿特别适合你"
```

## 第三部分：任务叙事系统

### 3.1 任务与玩家故事

```
任务系统的终极目标: 让玩家讲出自己的故事

设计师不写故事,而是创造"可能发生冲突的情境":
  └── 护送商队→遭遇山贼→选择: 战斗/交钱/谈判
  └── 山贼首领说"我们是活不下去才落草的"
  └── 你调查发现→当地官员横征暴敛→逼民为盗
  └── 你选择: 剿灭山贼(拿赏金)/帮助山贼(告发官员)/视而不见

每个选择分支都是玩家在"写故事"的笔触。

三个玩家,同一个任务起点,三个完全不同的故事:
  玩家A: 剿灭→赏金→完事→"今天赚了500金"
  玩家B: 调查→告发贪官→贪官被免→村民感激→"今天我为一个村子讨回了公道"
  玩家C: 加入→成为山贼首领→扩大势力→被招安成为将军→"今天我开始了从山贼到将军的传奇"

任务系统的价值不在于"设计了多少个任务"
而在于"每个任务能产生多少个不同的故事"
```

### 3.2 任务叙事的四个层次

```
层次1: 功能性叙事 (所有动态任务)
  "护送商队从A到B→奖励X金"
  故事: 无(功能性的)

层次2: 角色叙事 (涉及NPC关系的任务)
  "护送李商人→他是个有趣的老人→路上讲了他年轻时的冒险→到了B城他请你去他家做客→你见到了他的女儿..."
  故事: 简单的角色故事

层次3: 道德叙事 (有道德选择的任务)
  "护送途中被山贼劫→山贼首领是李商人的老朋友→他们曾是合伙人→因为商业纠纷反目→现在一个是富商一个是山贼→你帮谁?"
  故事: 道德困境+人物关系

层次4: 史诗叙事 (影响世界格局的任务)
  "护送任务的目的地正是合纵会盟的地点→因为你护送及时→齐国会盟成功→六国合纵抗秦→历史改变"
  故事: 个人行动→改变历史

设计师的追求:
  层次1的任务是填充(可无限生成)
  层次2-3的任务是骨架(每条路线有20-30个)
  层次4的任务是灵魂(每条路线只有1-2个，但影响深远)
```

# 音频技术标准

## 一、Wwise + UE5 集成

```
音频中间件: Wwise (Audiokinetic)
引擎集成: Wwise UE5 Plugin

Wwise项目结构:
  Actor-Mixer Hierarchy:
    ├── Ambient (环境音)
    ├── SFX (音效)
    │   ├── Combat (战斗)
    │   ├── Movement (移动)
    │   └── Interaction (交互)
    ├── VO (语音)
    ├── Music (音乐)
    └── UI (界面)

  Events: 所有可触发的音频事件
  States: 切换音乐状态(国家/昼夜/威胁度)
  Switches: 切换材质音效(草地/泥土/石板/木板)
  RTPCs: 实时参数(距离/速度/威胁度)
```

## 二、空间音频

```
3D定位:
  所有非UI音效使用3D定位
  衰减: 距离衰减+低通滤波(远处高频衰减)
  
  近距离(<5m): 完整频率
  中距离(5-30m): -6dB/倍距 + LowPass 8000Hz
  远距离(30-100m): -6dB/倍距 + LowPass 4000Hz
  极远(100m+): 不可闻(节省CPU)

  多普勒效应: 快速移动物体(箭矢/骑兵)开启

环境混响:
  室内: UE5 Audio Volume + Wwise Reverb
  小房间: Decay 0.3-0.5s
  大殿: Decay 1.5-2.5s
  洞穴/古墓: Decay 3-5s + 回声
  室外: 轻微混响(仅近距离反射)

闭塞/阻挡:
  墙壁阻挡: 音量-10~20dB + LowPass 2000Hz
  门隔音: 音量-15~30dB(取决于门的材质)
```

## 三、音频性能预算

```
同时播放Voice上限: 64

分配:
  环境音: 10-15 Voices
  战斗音效: 20-30 Voices (战斗中)
  语音: 5-8 Voices
  音乐: 4-8 Voices (Layer)
  UI: 5-8 Voices
  预留: 5-10 Voices

内存预算:
  总Audio Memory: <200MB (压缩后)
  环境音: 50MB
  战斗: 40MB
  语音: 50MB
  音乐: 40MB
  UI: 10MB
  预留: 10MB

CPU预算:
  总音频CPU: <2ms (音频专用线程)
  编码格式: Vorbis(压缩比高)/PCM(关键音效，低延迟)
```

## 四、音频管线

```
制作流程:
  1. 声音设计 → Wwise Authoring Tool
  2. 集成测试 → UE5 Editor (Wwise Profiler)
  3. 混音 → Wwise Mixer
  4. 最终输出 → SoundBank → 打包到游戏

SoundBank管理:
  按场景分包:
    Common.bnk (通用UI/交互音效)
    Level_STR.bnk (战略地图)
    Level_CIT.bnk (城池内部)
    Level_BAT.bnk (战场)
    Music.bnk (音乐)
    VO_[Lang].bnk (语音，按语言分包)

  加载策略:
    常驻: Common+UI (5MB)
    按场景加载: Level分包(10-15MB/场景)
    音乐: 流式播放(不常驻内存)
    语音: 按需加载(对话触发时)
```

## 五、音频QA

```
测试检查项:
  □ 所有UI按钮有声音反馈
  □ 兵器命中三种材质(肉体/盔甲/盾)声音不同
  □ 3D定位准确(听声辨位)
  □ 无爆音/削波(Digital Clipping)
  □ 无音频卡顿/掉声
  □ 不同天气下环境音切换正确
  □ Wwise Capture Log无报错
  □ 内存/CPU在预算内
  □ 5.1/7.1声道输出正确
  □ 耳机虚拟环绕正常

响度标准:
  使用Loudness Meter验证
  战斗场景: -18~-12 LUFS
  环境场景: -30~-20 LUFS
  峰值: <-1 dBTP
```

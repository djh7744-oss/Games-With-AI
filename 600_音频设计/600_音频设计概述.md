# 音频设计概述

## 音频愿景

> "让玩家听到战国。" 不仅仅是听到刀剑碰撞和马蹄声——而是听到风吹过函谷关的呼啸、临淄早市的喧嚣、长平战场四十万人的沉默。音频不是"附加在画面上的配乐"，而是战国世界本身的声音。

## 三大音频支柱

| 支柱 | 含义 | 检验标准 |
|------|------|---------|
| **沉浸感** | 环境音让玩家"感觉"到所处的地域和时代 | 闭眼能否分辨关中vs江南? |
| **反馈清晰** | 每个游戏动作都有明确的声音反馈 | 砍到肉/砍到盔甲/砍到盾——三种声音截然不同 |
| **情感驱动** | 音乐在关键时刻引导玩家情感 | 荆轲易水送别——音乐响起时玩家是否感到"不忍"? |

## 全局音频标准

| 项目 | 标准 |
|------|------|
| 音频引擎 | Wwise (中间件) + UE5 Native Audio |
| 采样率 | 48kHz (标准) / 96kHz (音乐录音) |
| 位深度 | 24bit (制作) / 16bit (最终压缩) |
| 输出格式 | 立体声(默认) / 5.1环绕声 / 7.1环绕声 / 耳机虚拟环绕 |
| 响度标准 | -23 LUFS (Integrated, EBU R128) |
| 动态范围 | 战斗场景 -18~-12 LUFS / 环境场景 -30~-20 LUFS |
| 最大峰值 | -1 dBTP (True Peak) |
| 音频预算 | 总内存<200MB(压缩后)/ 同时播放Voice<64 |

## 音频资产命名规范

```
格式: [类别]_[国家]_[名称]_[变体]

类别码:
  AMB = 环境音 (Ambient)
  SFX = 音效 (Sound Effect)
  VO = 语音 (Voice Over)
  MUS = 音乐 (Music)
  UI = 界面音效

示例:
  AMB_Qin_Xianyang_Market_Day     (咸阳市场白天环境)
  SFX_Sword_Hit_Flesh_01          (剑砍肉体音效01)
  VO_BaiQi_ChangPing_Order        (白起长平命令语音)
  MUS_Qi_LinZi_Theme              (齐·临淄主题音乐)
  UI_Button_Click_Wood            (木按钮点击)
```

## 音频优先级

```
P0 (不可中断):
  关键UI反馈/致命伤害警告/剧情语音

P1 (高优先):
  战斗音效(兵器碰撞/技能/Boss语音)
  紧急事件通知音

P2 (中优先):
  环境音效/脚步声/一般语音

P3 (低优先):
  环境细节(远处鸟鸣/风声)/氛围音

同优先级的Voice限制:
  P0/P1: 同时最多4个
  P2: 同时最多8个
  P3: 同时最多16个
```

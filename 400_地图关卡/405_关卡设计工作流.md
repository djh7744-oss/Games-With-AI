# 关卡设计工作流

## 一、关卡设计管线

### 从概念到可玩关卡

```
Phase 1: 概念设计(1-2天)
  输入: 设计文档中的关卡需求(如"需要一个山地伏击战场")
  输出: 
    - 关卡名称/主题
    - 参考图Mood Board(2-3张)
    - 关键玩法(这个关卡的核心体验是什么?)
    - 草图(Sketch): 俯视图+关键标注
  工具: 纸笔/Photoshop/PureRef

Phase 2: 白盒(Blockout)(2-5天)
  输入: 概念设计
  输出:
    - UE5中的白盒关卡(仅几何体，无美术)
    - 地形: Landscape(简单的Height Map)
    - 关键位置: 部署区/制高点/渡口/建筑群
    - 可游玩: 可以在白盒中跑动测试
  工具: UE5 Landscape + BSP/简单Static Mesh

Phase 3: 玩法验证(1-2天)
  输入: 白盒关卡
  输出:
    - 关卡设计师自己玩一遍(单人)
    - 邀请战斗设计师试玩
    - 验证: 尺寸是否合适? 关键点是否有效? 是否有趣?
    - 修改白盒

Phase 4: 美术打磨(5-10天)
  输入: 验证后的白盒
  输出:
    - 地形: 雕刻细节 + 贴图(Tileable Material)
    - 植被: 程序化植被放置(Procedural Foliage)
    - 建筑: 替换白盒为最终美术资产
    - 光照: 天空光/方向光/雾气
    - 后期: Post Process Volume
  工具: UE5 Landscape/Foliage/Megascans

Phase 5: 填充与优化(2-3天)
  输入: 美术打磨后的关卡
  输出:
    - 放置细节物件(石头/草木/道具)
    - LOD设置
    - 碰撞检测优化
    - 性能测试(千人战场→60FPS?)

Phase 6: 最终测试与发布(1-2天)
  输入: 优化后的关卡
  输出:
    - 全面测试(不同天气/不同时间/不同兵力规模)
    - Bug修复
    - 关卡文档更新
    - 关卡发布

总计: 约12-25个工作日/关卡
```

## 二、关卡设计标准

### 尺寸标准

```
战略大地图: 16km×14km
  1米=游戏中1单位

城池内部(Lv5都城): 1km×1km
  城墙内面积+城外缓冲区

战场(标准): 2km×2km
  部署区: 200m×战场宽度

村庄/探索点: 0.5km×0.5km
  小规模战斗/探索

关隘: 0.5km×1km(沿峡谷走向)
  狭窄通道+两端开放
```

### 性能标准

```
帧率目标:
  战略地图: 60FPS (RTX 3060)
  城池内部: 45-60FPS (NPC密度影响)
  千人战场: 60FPS (RTX 3060, 1440p)
  500人战场: 60FPS (GTX 1060, 1080p)

内存预算:
  战略地图: <4GB VRAM
  城池内部: <3GB VRAM
  战场: <5GB VRAM
  探索点: <2GB VRAM

加载时间:
  战略地图 ↔ 城池: <3秒(流式加载)
  战略地图 ↔ 战场: <5秒(预加载)
  快速旅行: <8秒(加载新区域)
```

## 三、关卡命名规范

```
关卡文件命名:
  LVL_[类型]_[名称]_[版本]
  
  类型:
    STR = 战略地图(Strategy)
    CIT = 城池(City)
    BAT = 战场(Battle)
    EXP = 探索(Exploration)
    INT = 室内(Interior)
    
  示例:
    LVL_STR_WarringStates_v03 (战略大地图)
    LVL_CIT_Xianyang_v02 (咸阳城)
    LVL_BAT_ChangPing_Plain_v01 (长平平原战场)
    LVL_EXP_GuiGu_v01 (鬼谷探索区域)
    LVL_INT_ChuTomb_v01 (楚王陵室内关卡)

关卡数据(JSON):
  存储: 地形参数/物件位置/关键点坐标/NPC生成点
  不存储: 美术资产(使用引用/软引用)
```

## 四、关卡检查清单

```
每个关卡提交前必须检查:

□ 尺寸: 是否在标准范围内?
□ 地形: 是否有可以利用的"关键地形"?
□ 部署区: 双方距离是否合适(不要太近也不要太远)?
□ 导航: 士兵AI能否正常寻路(NavMesh覆盖)?
□ 性能: 达到帧率目标了吗?
□ 碰撞: 是否有"卡住"或"穿过"的问题?
□ 照明: 所有区域都有光照(无纯黑区域)?
□ LOD: 远景物件有LOD吗?
□ 边界: 关卡边界有明确标识吗(玩家无法走出)?
□ 氛围: 这个关卡有"个性"吗(不是通用模板)?
```

## 五、关卡文档模板

```
每个关卡提交时附带:

## 关卡: [名称]
### 类型: [战场/城池/探索/室内]
### 尺寸: [X]×[Y]米
### 地形: [平原/山地/森林/...]
### 关键点:
  - [关键点1名称/坐标/占领效果]
  - ...
### 适合战斗规模: [小型/中型/大型]
### 天气可能: [晴天/雨天/...]
### 截图: (3张，不同角度)
### 特殊说明: (任何不同于标准的设计)
### 性能测试结果: (帧率/显存/加载时间)
```

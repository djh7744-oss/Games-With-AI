# 战场关卡设计

## 设计目标

> 对标《骑马与砍杀》的战场实例 +《全面战争》的规模感。每一场战斗发生在一个独立的战场关卡中——地形由战略地图上的当前位置决定。平原上的遭遇战、山地的伏击战、攻城时的城池战场，各有完全不同的关卡布局。

## 一、战场生成规则

```
战场由"战略地图坐标"决定:

1. 取战略地图上两军相遇的坐标
2. 查询该坐标的:
   - 基础地形(Biome+高度)
   - 附近特殊地貌(河流/森林/山地/村庄)
3. 根据参数生成战场关卡:
   - 尺寸: 1km×1km (小型) ~ 4km×4km (大型)
   - 地形: 高度图+植被+水体+建筑
   - 部署区: 双方各在关卡两端(东西或南北，取决于进入方向)

4. 如果附近有城池 → 生成攻城战场
   如果附近有关隘 → 生成关隘战场
   如果附近有河流 → 生成渡河战场
```

## 二、战场类型

### 1. 平原战场

```
尺寸: 2km×2km (标准)
地形: 平坦或轻微起伏
特征: 开阔视野/少量树木/可能有农田或村庄
适合: 大规模会战/骑兵冲锋
限制: 无特殊限制
出现: 中原/河北地区的遭遇战
```

### 2. 山地战场

```
尺寸: 1.5km×2km
地形: 高低差显著/岩石/稀疏植被
特征: 制高点决定胜负/弓兵优势
适合: 伏击战/防御战
限制: 骑兵受严重限制
出现: 太行山/秦岭/南方山区
```

### 3. 森林战场

```
尺寸: 1.5km×1.5km
地形: 密林覆盖/视线受阻
特征: 视野<50m/伏击概率极高
适合: 游击战/火攻
限制: 骑兵几乎无用/弓兵射程大减
出现: 楚地/巴蜀
```

### 4. 河岸/渡口战场

```
尺寸: 2km×1km
地形: 河流横亘/一侧高/一侧低
特征: 渡河方处于极度劣势
适合: 防御战
限制: 渡河方攻击-30%/需要抢占渡口
出现: 黄河/长江/淮河沿岸
```

### 5. 攻城战场

```
尺寸: 2km×2km(城池在中心偏守方一侧)
特殊: 包含完整的城池3D模型
部署: 攻方在城外(距离城墙500m)，守方在城墙上+城内
目标: 攻方: 突破城墙/城门 → 占领官衙
      守方: 击退攻方/坚守至援军到达

详见 [113_攻城战系统]
```

### 6. 村庄战场

```
尺寸: 1km×1km
地形: 小村庄+周围农田
特征: 房屋可提供掩护
适合: 小队战斗/剿匪
限制: 大规模军团无法展开
出现: 村庄附近的遭遇战
```

## 三、战场要素

### 部署区

```
双方部署区:
  位置: 战场两端(相向)
  尺寸: 200m×战场宽度
  特征: 平地/后方无障碍
  时间: 部署阶段30秒 → 玩家可调整初始位置和阵型
```

### 关键地形点

```
每个战场自动标记"关键点":

高地(制高点):
  Buff: 占领后全军视野+30%/弓兵射程+20%
  标记: 金色旗帜图标
  
渡口/桥梁:
  Buff: 控制渡口=控制敌方必须经过的路线
  标记: 蓝色桥梁图标
  
村庄/建筑群:
  Buff: 提供掩体/弓兵射界不受限
  标记: 灰色房屋图标
  
森林边缘:
  Buff: 隐蔽/伏击可能
  标记: 绿色树木图标

关键点可被夺取:
  派兵驻守 → 一段时间后占领 → 获得Buff
  被敌方夺取 → Buff转移
```

### 战场天气

```
进入战场时根据战略地图时间和季节 → 随机天气:

晴天(45%): 标准
多云(25%): 略微影响远程
雨天(15%): 弓弩射程-30%/火攻无效
雾天(8%): 视野<100m/伏击+30%/容易迷路
雪天(5%): 移速-40%/非耐寒部队士气-5
暴风雨(2%): 极端恶劣/水战翻船风险
```

## 四、千人战场性能优化

```
LOD策略(骑马与砍杀式):

近景(0-50m, ~20-50个士兵):
  完整3D模型+完整动画+完整AI(每1-2秒决策)+碰撞检测
  → GPU负载: 高

中景(50-150m, ~50-200个士兵):
  简化3D模型(LOD1)+简化动画(关键帧)+简化AI(每5秒决策)+简化碰撞
  → GPU负载: 中

远景(150-500m, ~200-800个士兵):
  极简3D模型(LOD2)+VAT(顶点动画纹理)+编队级AI(每10秒)+无碰撞
  → GPU负载: 低

极远(500m+, 其余士兵):
  群体渲染(一个"编队"模型代表大量士兵)
  → GPU负载: 极低

UE5 Mass Entity:
  远景士兵使用Mass Entity → 高效处理千人级群体
  仅近景士兵使用独立Actor+完整AI

性能目标:
  千人战场: 60FPS (RTX 3060/1440p)
  五百人战场: 60FPS (GTX 1060/1080p)
```

## 五、战后战场状态

```
战斗结束后 → 战场关卡卸载
但战场数据保留:

战场痕迹(可选保留):
  如果同一地点短期内再次发生战斗 → 
  战场上保留前次战斗的痕迹(残骸/血迹/兵器散落)
  → 视觉叙事: "这里曾经发生过一场大战"

战后时间推进:
  战斗时长 × 时间流速(通常1:6)
  → 一场30分钟的战斗=游戏中3小时
  → 回到战略地图时，时间已经推进了
```

# 地图关卡系统

## 设计目标

> 三张地图，三个尺度，一个无缝的战国世界。战略大地图让你俯瞰天下大势；城池内部让你感受市井烟火；战场关卡让你亲历千军万马。三张地图不是割裂的——从战略地图上的一次遭遇，加载进入战场；从战场取胜后，回到某座城池的酒馆庆祝。

## 三层地图架构

```
                    战国世界
                       │
        ┌──────────────┼──────────────┐
        ▼              ▼              ▼
   【战略层】      【城市层】      【战场层】
   鸟瞰大地图      城内自由探索    战斗实例关卡
   1:10000         1:1             1:1
   全中国可见      进入城池后      战斗触发时
   
   对标:            对标:            对标:
   三国群英传      GTA V           骑马与砍杀
   英雄无敌     大镖客2 NPC生态  + 全面战争
```

## 地图规模

| 方案 | 尺寸 | 城池 | 据点 | 适合阶段 | UE5方案 |
|------|------|------|------|---------|--------|
| A完整版 | 64×64km | 120+ | 200+ | AAA大团队 | World Partition |
| **B标准版** | 16×14km | 40+主城 | 80+县/关隘 | 独立团队 | World Partition |
| C最小版 | 8×8km | 15主城 | 20据点 | 原型验证 | 单Level |

> 建议: 先方案C做Phase 1-3，方案B做正式版，后续扩展至方案A。

## 三层地图技术衔接

```
战略地图 → 城池:
  触发: 点击城池图标 → 确认进入
  加载: Level Streaming异步加载城内关卡
  内容: 城池3D模型 + 城内NPC + 商铺数据 + 天气同步
  时间: <3秒(流式加载，预加载相邻城池)

战略地图 → 战场:
  触发: 两军相遇或主动进攻
  加载: 生成战场实例关卡(基于当前坐标+地形)
  内容: 地形 + 双方军队 + 战场物件
  时间: <5秒(预加载通用战场资产)

城池 → 战略地图:
  触发: 走出城门/选择"离开城池"
  卸载: 城内关卡卸载，战略地图恢复

战场 → 战略地图:
  触发: 战斗结束
  卸载: 战场关卡卸载
  同步: 战斗消耗时间在实际游戏时间中流逝
```

## 设计支柱

| 支柱 | 含义 | 检验标准 |
|------|------|---------|
| **可读性** | 玩家在鸟瞰视角下能快速理解地图信息 | 5秒内识别: 边界/己方/最近城池/敌军 |
| **探索感** | 战争迷雾+隐藏地点带来"发现"的快感 | 首次揭开迷雾时是否有期待感? |
| **地域差异** | 不同区域视觉和玩法特征鲜明 | 关中vs江南，凭视觉可区分 |
| **战略深度** | 地形关隘真正影响军事决策 | 函谷关是否真有"一夫当关"感? |
| **沉浸感** | 城内NPC生态让城池"活"起来 | 玩家是否会在酒馆坐下"听"NPC聊天? |
| **性能** | 千人战场不掉帧，大地图不卡顿 | RTX3060/1440p/60FPS |

## 与各系统的交互

| 关联系统 | 交互方式 | 影响 |
|---------|---------|------|
| 战斗系统 | 战场关卡是战斗的物理空间 | 地形决定战术 |
| 经济系统 | 城池是贸易节点 | 商路在地图上可见 |
| AI系统 | NPC在城池/地图上活动 | NPC生态→地图上的人口流动 |
| 任务系统 | 任务目标在地图上标记 | 地图是任务的导航 |

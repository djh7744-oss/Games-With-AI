## 三、代码规范

```
C++ 规范:
  - 遵循 UE5 Coding Standard
  - 类前缀: U(Actor派生)/A(Actor)/F(结构体)/E(枚举)/I(接口)
  - 命名: PascalCase
  - 禁止: 全局变量/裸指针(用TObjectPtr)/魔法数字

Blueprint 规范:
  - 蓝图仅用于: UI/脚本/快速原型
  - 复杂逻辑 → C++实现，蓝图调用
  - 蓝图命名: BP_[类型]_[名称]
  - 避免: 超长蓝图线(>20节点 → 考虑C++)

Git 规范:
  - 主分支: main(发布) / develop(开发)
  - Feature分支: feature/[系统]-[描述]
  - Commit格式: [模块] 简要描述
    例: "[Combat] 修复四方向攻击碰撞检测"
  - LFS: >100MB文件使用Git LFS
```
